= Antora documentation site for Quarkus LangChain4j

To edit the documentation with live-reload, start Quarkus in dev mode

[source,shell]
----
$ cd docs
$ mvn quarkus:dev
----

Once the application has started, hit `w` to open the freshly generated site in browser.

See https://docs.quarkiverse.io/quarkus-antora/dev/index.html[Quarkus Antora] documentation for more details.

== Regenerating diagrams

Mermaid source files live in `diagrams/` and render to SVG in `modules/ROOT/assets/images/`. The SVGs are generated files: never edit them directly. After editing a `.mmd` file, regenerate all SVGs:

[source,shell]
----
$ cd docs
$ npm run diagrams
----

The script (`diagrams/generate.sh`) renders with `diagrams/mermaid-config.json`, which sets `useMaxWidth: false` so each SVG gets an explicit pixel `width`/`height` instead of `width="100%"`. Without that, browsers treat the SVG as having no intrinsic width and stretch it to the full width of the docs column, so narrow diagrams appear disproportionately large. The site stylesheet's `max-width: 100%` still scales diagrams wider than the column down to fit.

Commit both the updated `.mmd` source and the regenerated `.svg` output.
